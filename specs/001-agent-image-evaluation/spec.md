# Feature Specification: Agent图像评估系统

**Feature Branch**: `001-agent-image-evaluation`

**Created**: 2026-05-18

**Status**: Draft

**Input**: User description: "创建一个应用，提供agent评估能力
1. 图片生成的评估，评估生成后的图片是否满足prompt要求。
   1.1 openclip apple/DFN2B-CLIP-ViT-L-14-39B 评估，评估语义一致性
   1.2 使用 comfyui qwen_vl 工作流对生成的图做结构化描述，然后使用 LLM 跟原始 prompt 进行对比
2. 评估任务类似comfyui api那样，先提交评估任务，然后返回任务id，然后通过 ws 跟踪评估进度和结果
3. 提供前端页面，可以查看历史评估结果"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 提交图像评估任务 (Priority: P1)

用户上传一张AI生成的图片和对应的prompt描述，系统接收任务并返回任务ID供后续查询。

**Why this priority**: 这是核心功能入口，用户必须能够提交评估任务才能使用整个系统。

**Independent Test**: 可通过API直接提交任务并验证返回的任务ID，交付可追踪的评估句柄。

**Acceptance Scenarios**:

1. **Given** 用户已准备好图片文件和对应的prompt描述，**When** 用户通过API提交评估任务（包括图片URL/路径和prompt文本），**Then** 系统返回唯一任务ID和初始状态"pending"
2. **Given** 用户提交了不包含图片或prompt的任务，**When** 系统处理请求，**Then** 返回参数缺失的错误提示
3. **Given** 用户提交任务后，**When** 系统开始处理，**Then** 任务状态变为"processing"

---

### User Story 2 - 实时追踪评估进度 (Priority: P1)

用户通过WebSocket连接实时接收评估任务的状态更新和最终结果。

**Why this priority**: 评估任务可能耗时较长，用户需要实时了解进度以规划后续工作。

**Independent Test**: 可通过模拟WebSocket连接验证任务状态推送功能，交付实时反馈能力。

**Acceptance Scenarios**:

1. **Given** 用户已获得任务ID，**When** 用户建立WebSocket连接并订阅指定任务，**Then** 用户收到任务当前状态的实时推送
2. **Given** 评估任务正在进行中，**When** 每个评估阶段完成，**Then** 用户收到进度更新消息（包含完成的阶段和当前阶段）
3. **Given** 评估任务完成，**When** 系统推送最终结果，**Then** 结果包含语义一致性评分、结构化描述、LLM对比分析
4. **Given** 评估任务执行失败，**When** 系统推送错误信息，**Then** 包含错误类型和可读的失败原因

---

### User Story 3 - 查看历史评估记录 (Priority: P2)

用户通过前端页面查看所有历史评估任务及其结果。

**Why this priority**: 用户需要回顾和对比历史评估结果，以便分析模型表现和优化方向。

**Independent Test**: 可通过前端页面查看任务列表，交付可追溯的评估历史管理能力。

**Acceptance Scenarios**:

1. **Given** 用户已有多条评估记录，**When** 用户打开历史记录页面，**Then** 页面展示任务列表，按提交时间倒序排列
2. **Given** 用户查看任务列表，**When** 每条记录展示关键信息，**Then** 显示任务ID、提交时间、状态、综合评分和prompt摘要
3. **Given** 用户需要筛选记录，**When** 用户使用筛选条件（如日期范围、评分区间），**Then** 列表仅显示符合条件的结果

---

### User Story 4 - 查看评估详情 (Priority: P2)

用户点击某条历史记录查看完整的评估报告。

**Why this priority**: 用户需要详细分析每次评估的各个维度指标以深入理解结果。

**Independent Test**: 可通过点击列表项查看详情页面，交付完整的评估洞察能力。

**Acceptance Scenarios**:

1. **Given** 用户在历史列表中点击某条记录，**When** 系统加载评估详情，**Then** 展示原图、prompt原文、各维度评分和详细分析
2. **Given** 用户查看评估详情，**When** 查看语义一致性评分，**Then** 展示CLIP模型评分及与基准的对比说明
3. **Given** 用户查看评估详情，**When** 查看LLM分析结果，**Then** 展示结构化描述、对比结论和一致性判断

---

### Edge Cases

- 评估过程中图片文件被删除或无法访问：任务状态标记为"failed"，推送错误通知
- WebSocket连接在评估过程中断开：用户重新连接后可获取最新状态，历史数据通过REST API补充
- 用户提交的图片格式不支持：返回格式错误提示，支持格式列表在API文档中说明
- 评估服务不可用（如模型加载失败）：任务进入"queued"状态等待重试或标记为"failed"
- 并发提交大量评估任务：系统需支持任务队列管理，按提交顺序执行

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统 MUST 提供REST API用于提交评估任务，输入包括图片（URL或base64）和prompt文本
- **FR-002**: 提交任务API MUST 返回唯一的任务ID和初始状态"pending"
- **FR-003**: 系统 MUST 提供WebSocket端点，支持用户订阅特定任务的实时进度和结果
- **FR-004**: 评估任务 MUST 包含两个评估维度：CLIP语义一致性评分和LLM结构化对比分析
- **FR-005**: CLIP评估 MUST 使用指定模型（apple/DFN2B-CLIP-ViT-L-14-39B）计算图片与prompt的语义相似度
- **FR-006**: LLM评估 MUST 先用Qwen VL生成图片结构化描述，再与原始prompt进行对比分析
- **FR-007**: 评估结果 MUST 包含各维度评分（0-100或0-1范围）、综合评分和分析结论
- **FR-007a**: CLIP语义一致性评分阈值定义：≥0.7表示语义一致，≤0.5表示语义不一致，0.5-0.7为模糊区间需人工复核
- **FR-008**: 系统 MUST 提供REST API用于查询任务状态和获取评估结果（通过任务ID）
- **FR-009**: 系统 MUST 提供前端页面用于查看历史评估记录列表，支持分页和基本筛选（按日期范围、任务状态）
- **FR-009a**: 系统 MUST 支持创建"项目/批次"来分组管理评估任务，FR-009的历史列表支持按项目筛选；两者为互补功能，FR-009a提供组织能力
- **FR-010**: 前端页面 MUST 展示评估详情，包括原图、各维度评分、详细分析内容
- **FR-011**: 评估任务状态 MUST 包含：pending（待处理）、queued（排队中）、processing（处理中）、completed（已完成）、failed（失败）
- **FR-012**: 系统 MUST 持久化存储评估任务记录，包含输入数据、评估结果和时间戳

### Key Entities

- **评估任务（EvaluationTask）**: 代表一次完整的图像评估操作，包含唯一标识符、输入数据（图片URL、prompt）、任务状态、创建时间、完成时间、所属项目ID
- **评估结果（EvaluationResult）**: 代表某次评估的输出，包含语义一致性评分、LLM分析结果、结构化描述文本、综合评分
- **评估历史记录（HistoryRecord）**: 代表前端展示用的历史条目，包含任务摘要信息用于列表展示
- **项目/批次（Project）**: 用于分组管理评估任务，用户可创建项目来组织相关的评估任务，便于批量查看和对比分析

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 用户提交评估任务后，系统在5秒内返回任务ID
- **SC-002**: 单张图片评估任务在60秒内完成并推送结果
- **SC-003**: 评估结果的语义一致性评分范围为0-1，0表示完全不匹配，1表示完全匹配
- **SC-004**: 历史记录列表支持至少1000条记录的分页加载，加载下一页延迟不超过2秒
- **SC-005**: WebSocket连接建立后，状态更新推送延迟不超过1秒
- **SC-006**: 前端页面在网络正常情况下可正常访问，图片加载成功率不低于99%

---

## Assumptions

- 图片格式：假设支持常见格式（PNG、JPG、JPEG、WebP），超出范围时返回明确错误提示
- 图片来源：假设图片通过URL或Base64编码方式提供，本地文件上传作为后续增强功能
- 并发能力：假设系统需支持至少10个并发评估任务，后续可根据负载情况横向扩展
- 数据保留：评估历史记录默认保留30天，可配置，具体由运营需求决定
- Docker部署：Docker配置为可选功能，当需要容器化部署时实现；核心功能不依赖Docker
- LLM服务：假设使用项目已有的LLM服务接口或云端API
- 认证机制：系统无需用户认证，任何人可提交评估任务，通过任务ID管理任务
- 日志要求：记录每个评估阶段的输入输出、耗时、模型响应，支持问题排查和模型性能分析
